# Epic 2: Output System

Implement the complete output system with multiple formatters and progress indicators.

## Background
The CLI needs to support multiple output formats for different use cases:
- **Plain**: Default, human-readable with colors and formatting
- **JSON**: Structured for AI agents and scripting (via `--json` flag)
- **Table**: Tabular view for lists (via `-f table`)
- **CSV**: For spreadsheet import (via `-f csv`)
- **NDJSON**: Newline-delimited JSON for streaming large lists

## Design Philosophy

**Human-readable by default, JSON opt-in.** While this CLI is designed for AI agent consumption, humans also use it interactively. Following the pattern from raindrop-cli:

```bash
# Default: Human-readable
monarch auth status
# ✓ Authenticated
#   Backend: file

# Opt-in: JSON for scripts/agents
monarch auth status --json
# {"authenticated": true, "storage_backend": "file", ...}
```

This approach:
- Provides a good out-of-box experience for humans
- Keeps AI/scripting use cases fully supported via `--json`
- Follows clig.dev guidelines for CLI design

## Scope
- Full output formatters (expand bootstrap version from Phase 1)
- Progress indicators (spinners for long operations)
- TTY detection for smart defaults
- Color handling with NO_COLOR support

## I/O Contract (from clig.dev guidelines)
| Stream | Purpose | Examples |
|--------|---------|----------|
| **stdout** | Primary data output | JSON, CSV (when requested) |
| **stderr** | Human output, progress, errors | Plain text, spinners, Rich formatting |

This separation allows piping: `monarch accounts list --json | jq '.[] | .balance'`

## Response Time
Print something to stderr within 100ms, especially before network I/O. Users shouldn't think the CLI is frozen.

## TTY Detection
| Context | Behavior |
|---------|----------|
| TTY (interactive) | Colors, spinners, prompts OK |
| Non-TTY (piped/scripted) | No colors, no spinners, no prompts |

## Command Flag Patterns

Simple status commands:
- `--json`: Output as JSON (for scripts/AI agents)

List/data commands:
- `--json`: Output as JSON
- `-f, --format`: Choose format (table, csv, plain)
- `--ndjson`: Stream as newline-delimited JSON
- `--raw`: Output raw API response without transforms
- `--no-color`: Disable colors (also respects NO_COLOR env)
