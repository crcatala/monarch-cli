# Epic 5: Testing & Documentation

Comprehensive testing and documentation to complete v1.0.

## Background
With all features implemented, this phase ensures quality through testing and provides documentation for users.

## Testing Strategy

### Unit Tests
- Test transformers with mock API responses
- Test output formatters with various data types
- Test date utilities for all presets
- Test config loading from environment

### CLI Tests with CliRunner
- Test each command with mocked services
- Test error handling paths
- Test output format variations
- Test argument validation

### Schema Contract Tests
Ensure output schemas remain stable for AI agents:
```python
def test_account_schema_has_required_fields():
    result = transform_account(raw_account)
    assert "id" in result
    assert "name" in result
    assert "balance" in result
```

### Live Tests (Local Development Only)
- NOT run in CI
- Require real authentication
- Verify actual API integration
- Guard with MONARCH_LIVE_TESTS=1
