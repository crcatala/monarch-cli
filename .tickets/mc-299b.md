# Full Output Formatters

Expand the bootstrap output module to support all output formats.

## Location
`src/monarch_cli/output/__init__.py` (replace bootstrap version)

## Design Philosophy

**Human-readable by default, JSON opt-in.** While this CLI is designed for AI agent consumption, humans also use it interactively. The default output should be readable in a terminal, with `--json` flag for machine-readable output.

This follows the pattern from raindrop-cli and clig.dev guidelines:
- Default: Styled, human-readable output (colors, formatting)
- `--json` flag: Structured JSON for scripts and AI agents
- Data to stdout, messages/progress to stderr

## Output Formats

### Plain (default)
Human-readable styled output with Rich formatting.
```python
console.print("[green]✓ Success[/green]")
console.print(f"  Items: {count}")
```

### JSON (via --json flag)
Pretty-printed JSON with 2-space indentation.
```python
print(json.dumps(data, indent=2, default=str))
```

### Table (via -f table, for list data)
Rich tables for human reading of lists/collections.
```python
from rich.table import Table
table = Table()
for key in items[0].keys():
    table.add_column(key)
for item in items:
    table.add_row(*[str(v) for v in item.values()])
console.print(table)
```

### CSV (via -f csv)
Standard CSV output for spreadsheet import.
```python
import csv
from io import StringIO
writer = csv.DictWriter(output, fieldnames=items[0].keys())
writer.writeheader()
writer.writerows(items)
```

### NDJSON (via --ndjson)
Stream items one per line for large datasets.
```python
for item in data:
    print(json.dumps(item, default=str))
```

## Command Flag Patterns

For simple status commands (auth status, auth ping):
```bash
monarch auth status          # Human-readable (default)
monarch auth status --json   # JSON output
```

For list/data commands (accounts list, transactions list):
```bash
monarch accounts list              # Human-readable plain (default)
monarch accounts list --json       # JSON output
monarch accounts list -f table     # Table format
monarch accounts list -f csv       # CSV format
```

## Raw Mode
When `--raw` is specified, skip all transformation and output the API response directly. Useful for debugging or when users need fields not in our transforms.

## Empty Results
For empty lists:
- Plain: "No results found." message
- JSON: `[]`
- Table: "No results" message
- CSV: Headers only (no rows)
- NDJSON: No output

## default=str Handler
Use `default=str` in json.dumps to handle datetime objects and other non-serializable types.
