# Epic 4: AI Agent Optimization

Implement features specifically for AI agent consumption and batch operations.

## Background
This CLI is designed for AI agent integration. While the core commands work well for agents, this phase adds features that make agent workflows more efficient:

1. **Quiet Mode**: Minimal output (IDs only) for when agents just need identifiers
2. **Batch Operations**: Update multiple transactions efficiently

## Scope

### Quiet Mode (`--quiet, -q`)
Output only IDs, one per line:
```bash
monarch accounts list --quiet
# Output:
ACC123
ACC456
ACC789
```

Useful for:
- Passing IDs to other commands
- Scripting with xargs
- Minimal parsing overhead

### Batch Operations
Update multiple transactions at once:
```bash
echo -e "TXN001\nTXN002" | monarch transactions batch-update --stdin --category CAT123
```

Features:
- Read transaction IDs from stdin
- Concurrency control (--max-concurrency)
- Dry run support
- Progress reporting

## Why These Features Matter for AI Agents

### Quiet Mode
- Reduces token usage when processing output
- Simple parsing (just split by newlines)
- Composable with shell pipelines

### Batch Operations
- Single API authentication instead of per-transaction
- Controlled concurrency prevents rate limiting
- Progress feedback for long operations

## Priority
P1 (v1.0) - Important for AI agent use case but not blocking MVP

## Acceptance Criteria

- [ ] `--quiet` mode available on list commands
- [ ] Quiet output is IDs only, one per line
- [ ] `batch-update` command reads from stdin
- [ ] Concurrency control with --max-concurrency
- [ ] Batch operations support --dry-run
- [ ] Progress shown for batch operations
