# Categories Commands

Implement categories CLI command: list.

## Location
`src/monarch_cli/commands/categories.py`

## Purpose
Category IDs are needed for transaction updates. This command lets users look up category IDs by name.

## Command

### `monarch categories list`
List all transaction categories with their IDs.

```python
@app.command("list")
@handle_errors
def list_categories(
    format: OutputFormat = typer.Option(OutputFormat.JSON, "--format", "-f")
):
    """List all transaction categories."""
    with spinner("Fetching categories..."):
        client = get_authenticated_client()
        data = run_async(client.get_transaction_categories())

    # Flatten nested category structure
    result = []
    for group in data.get("categories", []):
        for cat in group.get("children", []):
            result.append({
                "id": cat.get("id"),
                "name": cat.get("name"),
                "group": group.get("name"),
                "icon": cat.get("icon"),
            })

    output(result, format)
```

## Output Schema
| Field | Type | Description |
|-------|------|-------------|
| id | string | Category ID (for API calls) |
| name | string | Category name |
| group | string | Parent group name |
| icon | string | Icon identifier |

## Category Structure
Monarch Money organizes categories in groups:
- Income
  - Paycheck
  - Interest
  - ...
- Food & Drink
  - Groceries
  - Restaurants
  - ...
- Housing
  - Rent
  - Mortgage
  - ...

The transformation flattens this hierarchy, including the group name for context.

## CLI Examples
```bash
# List all categories
monarch categories list
monarch categories list -f table

# Find category ID by name
monarch categories list | jq '.[] | select(.name == "Groceries")'
monarch categories list | jq '.[] | select(.group == "Food & Drink")'

# Use in transaction update
GROCERIES_ID=$(monarch categories list | jq -r '.[] | select(.name == "Groceries") | .id')
monarch transactions update TXN123 --category $GROCERIES_ID
```

## Why Flattened Output?
- Easier to search/filter with jq
- Table output works directly
- CSV export makes sense
- Original hierarchy available via --raw
