# Auth Commands Implementation

Implement all authentication commands: login, status, logout, doctor, ping, setup.

## Location
`src/monarch_cli/commands/auth.py`

## Commands Overview

### `monarch auth login`
Interactive login to Monarch Money with MFA support.
- Prompts for email and password (using getpass for password)
- Handles RequireMFAException and prompts for MFA code
- Asks user to choose storage backend (keyring or file)
- Saves token and shows account count on success
- Uses `--storage` flag to skip interactive storage choice

### `monarch auth status`
Check authentication status.
- Returns JSON with authenticated boolean, storage_backend, and message
- Uses @handle_errors decorator
- Supports --format flag

### `monarch auth logout`
Remove stored authentication token.
- Clears from all backends by default
- Optional `--storage` flag to clear specific backend
- Resets cached client

### `monarch auth doctor`
Diagnose environment and auth storage.
- Checks keyring availability and backend type
- Shows status of all token storage locations
- Tests API connectivity if authenticated
- Uses rich console for formatted output

### `monarch auth ping`
Test basic API connectivity.
- Requires authentication
- Makes simple API call (get_accounts)
- Returns {status: "ok", message: ...} on success
- Uses @handle_errors decorator

### `monarch auth setup`
Show setup instructions.
- Displays formatted help text with examples
- Explains storage options
- Points to troubleshooting commands

## Security Notes
- Never accept passwords via command-line flags
- Use getpass for password input (no echo)
- Token is never displayed in output

## MFA Flow
```python
try:
    run_async(mm.login(email, password, save_session=False))
except RequireMFAException:
    mfa_code = input("MFA Code: ")
    run_async(mm.multi_factor_authenticate(email, password, mfa_code))
```

## CLI Examples
```bash
monarch auth login               # Interactive login
monarch auth login -s file       # Login with file storage
monarch auth status              # Check auth state
monarch auth status -f compact   # Compact JSON output
monarch auth logout              # Clear all tokens
monarch auth logout -s keyring   # Clear only keyring
monarch auth doctor              # Diagnose issues
monarch auth ping                # Test connectivity
monarch auth setup               # Show instructions
```
