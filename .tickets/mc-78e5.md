# Account Transformer

Create transformer to convert account API responses to CLI-friendly format.

## Location
`src/monarch_cli/transformers/accounts.py`

## Why Transformers?
The raw API response has nested structures and inconsistent naming. Transformers provide:
1. Stable, documented output schema for AI agents
2. Flattened structure for easy consumption
3. Consistent field names (snake_case)
4. Protection against API changes

## Implementation
```python
from typing import Any

def transform_account(raw: dict[str, Any]) -> dict[str, Any]:
    """Transform a single account."""
    return {
        "id": raw.get("id"),
        "name": raw.get("displayName"),
        "type": raw.get("type", {}).get("display"),
        "subtype": raw.get("subtype", {}).get("display"),
        "balance": raw.get("currentBalance"),
        "institution": raw.get("institution", {}).get("name"),
        "is_active": not raw.get("isHidden", False),
        "is_manual": raw.get("isManual", False),
        "last_updated": raw.get("updatedAt"),
    }

def transform_accounts(raw: dict[str, Any]) -> list[dict[str, Any]]:
    """Transform accounts API response."""
    return [transform_account(acc) for acc in raw.get("accounts", [])]
```

## Output Schema
| Field | Type | Description |
|-------|------|-------------|
| id | string | Account ID |
| name | string | Display name |
| type | string | Account type (Checking, Savings, etc.) |
| subtype | string | Account subtype |
| balance | float | Current balance |
| institution | string | Bank/institution name |
| is_active | bool | True if not hidden |
| is_manual | bool | True if manually added |
| last_updated | string | ISO timestamp |

## Schema Stability
These fields are committed API - changes would be breaking for AI agents. Add new fields cautiously, never remove without deprecation.

## Null Handling
Use `.get()` with defaults to handle missing fields gracefully. The `--raw` flag allows access to original data if needed.
