# Quiet Mode (--quiet Flag)

Implement --quiet flag for minimal output (IDs only).

## Purpose
AI agents often just need identifiers, not full records. Quiet mode reduces output to bare essentials.

## Implementation

### Update output module
```python
def output(
    data: Any,
    format: OutputFormat = OutputFormat.JSON,
    ndjson: bool = False,
    raw: bool = False,
    quiet: bool = False,  # NEW
    id_field: str = "id",  # NEW - which field to extract
) -> None:
    if quiet:
        if isinstance(data, list):
            for item in data:
                if isinstance(item, dict) and id_field in item:
                    print(item[id_field])
                else:
                    print(item)
        elif isinstance(data, dict) and id_field in data:
            print(data[id_field])
        return
    # ... rest of existing logic
```

### Update commands
Add `--quiet, -q` option to list commands:
```python
@app.command("list")
@handle_errors
def list_accounts(
    format: OutputFormat = typer.Option(OutputFormat.JSON, "--format", "-f"),
    quiet: bool = typer.Option(False, "--quiet", "-q", help="Output IDs only"),
    ...
):
    ...
    output(result, format, quiet=quiet)
```

## Output Examples
```bash
# Normal output
monarch accounts list -f compact
[{"id": "ACC123", "name": "Checking", ...}, ...]

# Quiet output
monarch accounts list --quiet
ACC123
ACC456
ACC789
```

## Use Cases
```bash
# Count accounts
monarch accounts list -q | wc -l

# Refresh specific accounts
monarch accounts list -q | head -3 | xargs -I {} monarch accounts refresh -a {}

# Pass to jq for further processing
for id in $(monarch accounts list -q); do
    echo "Processing $id"
done
```

## Interaction with Other Flags
- `--quiet` takes precedence over `--format`
- `--quiet` takes precedence over `--ndjson`
- `--raw` is incompatible with `--quiet` (error if both specified)
