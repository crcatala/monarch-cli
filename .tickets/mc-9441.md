# Dev Tooling & Verification Scripts

Set up development tooling and verification scripts for consistent quality checks.

## Verification Script Options

### Option A: Parallel Runner (Recommended)
Use Armin Ronacher's `run-parallel.py` script for parallel execution with live status:
```bash
curl -o scripts/run-parallel.py https://gist.githubusercontent.com/mitsuhiko/a4b6b70e96c8075b92a4de00b340cc52/raw
chmod +x scripts/run-parallel.py
```

Create `scripts/verify.py`:
```python
#!/usr/bin/env -S uv run --script
import subprocess, sys
result = subprocess.run([
    "uv", "run", "scripts/run-parallel.py", "--fail-fast",
    "format", "uv run ruff format --check .",
    "lint", "uv run ruff check .",
    "typecheck", "uv run mypy src/",
    "test", "uv run pytest -x",
])
sys.exit(result.returncode)
```

### Option B: Makefile (Simpler)
```makefile
.PHONY: verify test lint typecheck format
verify: format-check lint typecheck test
format-check: uv run ruff format --check .
lint: uv run ruff check .
typecheck: uv run mypy src/
test: uv run pytest
```

## CI Workflow (.github/workflows/ci.yml)
Basic workflow that runs on push/PR:
1. Checkout code
2. Install uv (astral-sh/setup-uv@v4)
3. `uv sync` to install deps
4. Format check, lint, typecheck, test (excluding live tests)
5. Upload coverage to codecov

## Why This Matters
- Coding agents should run `make verify` or `uv run scripts/verify.py` after every change
- Parallel execution = faster feedback loop
- Consistent checks in CI and local development
