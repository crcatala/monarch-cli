# Epic 0: Project Setup

Initialize the Monarch CLI project repository with proper Python tooling and structure.

## Background
Monarch CLI is a Python CLI wrapper around the `monarchmoneycommunity` library that enables AI agent integration with Monarch Money financial data. The community fork fixes critical issues in the upstream `monarchmoney` library including broken API endpoints, auth persistence, and budget queries.

## Scope
- Repository initialization with uv
- Complete pyproject.toml configuration with all dependencies and tool configs
- src-layout project structure following Python best practices
- Development tooling setup (ruff, mypy, pytest)
- Verification scripts for CI/CD

## Why This Matters
A solid foundation prevents technical debt and ensures consistent developer experience. The src-layout prevents import ambiguity, and having all tools configured upfront means every subsequent phase benefits from linting, type checking, and testing from day one.

## Key Decisions
- **Package manager**: uv (10-100x faster than pip, by Astral)
- **Linting/Formatting**: Ruff (single tool, 100x faster)
- **Type checking**: mypy with strict mode
- **Testing**: pytest with asyncio support
- **Project layout**: src-layout for proper package isolation

## Acceptance Criteria

- [ ] `uv run monarch --help` shows basic help text
- [ ] `uv run monarch --version` shows version 0.1.0
- [ ] `make verify` runs all checks (format, lint, typecheck, test)
- [ ] Directory structure matches the planned layout
- [ ] All dev dependencies installed and working
- [ ] .gitignore properly configured for Python
