# Session Management (Dual-Backend)

Implement secure, dual-backend session token storage with keyring and file fallback.

## Location
`src/monarch_cli/core/session.py`

## Storage Backends

| Storage | Location | Security | Use Case |
|---------|----------|----------|----------|
| **Keyring** (default) | OS credential store | ✅ Encrypted at rest | Recommended for most users |
| **Session file (JSON)** | ~/.config/monarch-cli/session.json | ⚠️ Plain file (0600 perms) | Portability, containers |
| **Session file (compat)** | ~/.mm/mm_session.pickle | ❌ Unsafe (pickle) | Library interop (opt-in only) |

## Key Functions

```python
class StorageBackend(str, Enum):
    KEYRING = "keyring"
    FILE = "file"
    FILE_COMPAT = "file-compat"  # Legacy pickle, opt-in only

def save_session_token(token: str, backend: StorageBackend) -> None
def get_session_token() -> str | None
def delete_session_token(backend: StorageBackend | None = None) -> None
def has_valid_session() -> bool
def get_storage_info() -> dict  # For auth status/doctor
```

## Token Load Precedence (highest to lowest)
1. MONARCH_TOKEN env var (best for CI/agents)
2. Keyring
3. JSON session file
4. Legacy compat pickle (if enabled)

## Security Requirements for File Storage
- Use JSON (not pickle) for default file storage
- Atomic writes via temp file + rename
- Set 0600 permissions (owner read/write only) BEFORE writing content
- Use tempfile.mkstemp() with os.fchmod() for secure creation

```python
fd, tmp_path = tempfile.mkstemp(dir=session_path.parent, suffix=".tmp")
try:
    os.fchmod(fd, 0o600)  # Set perms before writing
    with os.fdopen(fd, "w") as f:
        json.dump({"token": token}, f)
    os.replace(tmp_path, session_path)  # Atomic replace
except Exception:
    if os.path.exists(tmp_path):
        os.unlink(tmp_path)
    raise
```

## 💡 TIP FOR AGENT WORKFLOWS

After human completes auth login, the token is stored in keyring or session file.

For agent automation, the token can also be passed via environment variable:

```bash
# Human extracts token once (after login)
monarch auth status  # Shows storage location

# For keyring storage, extract with:
# python -c "import keyring; print(keyring.get_password('com.monarch-cli', 'monarch-token'))"

# For file storage:
# cat ~/.config/monarch-cli/session.json | jq -r .token

# Then set for agent sessions:
export MONARCH_TOKEN="<extracted-token>"
```

This allows agents to run in CI or fresh shells without re-authenticating.

**Security note**: Token in env var is visible in process listings. Acceptable for local dev, not for shared systems.
