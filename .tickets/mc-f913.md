# Transaction Transformer

Create transformer to convert transaction API responses to CLI-friendly format.

## Location
`src/monarch_cli/transformers/transactions.py`

## Implementation
```python
from typing import Any

def transform_transaction(raw: dict[str, Any]) -> dict[str, Any]:
    """Transform a single transaction."""
    return {
        "id": raw.get("id"),
        "date": raw.get("date"),
        "amount": raw.get("amount"),
        "description": raw.get("merchant", {}).get("name") or raw.get("plaidName"),
        "category": raw.get("category", {}).get("name"),
        "category_id": raw.get("category", {}).get("id"),
        "account": raw.get("account", {}).get("displayName"),
        "account_id": raw.get("account", {}).get("id"),
        "is_pending": raw.get("pending", False),
        "notes": raw.get("notes"),
    }

def transform_transactions(raw: dict[str, Any]) -> list[dict[str, Any]]:
    """Transform transactions API response."""
    results = raw.get("allTransactions", {}).get("results", [])
    return [transform_transaction(txn) for txn in results]
```

## Output Schema
| Field | Type | Description |
|-------|------|-------------|
| id | string | Transaction ID |
| date | string | Transaction date (YYYY-MM-DD) |
| amount | float | Amount (negative for expenses) |
| description | string | Merchant name or plaid name |
| category | string | Category name |
| category_id | string | Category ID (for updates) |
| account | string | Account display name |
| account_id | string | Account ID |
| is_pending | bool | True if pending |
| notes | string | User notes |

## Description Field Logic
Priority for description:
1. merchant.name (clean merchant name)
2. plaidName (raw from bank)

This gives the best available description while allowing `--raw` for full details.

## Amount Sign Convention
- Negative amounts = expenses/outflows
- Positive amounts = income/inflows
This matches Monarch Money's convention.

## Including IDs
Include both human-readable names AND IDs (category_id, account_id) so users can:
1. View: see readable names
2. Update: use IDs for API calls
